# CSS

css도 이미 아는 내용이 많기 때문에 몰랐던 내용을 위주로 빠르게 정리하고 넘어가겠다.

## a태그에 대해서 색을 빨갛게 칠해줘
```
<style>
    a {
        color: red;
    }
</style>
```

## 스타일 태그가 아닌 속성을 이용한 방식
```
<li><a href="2.html" style="color:red">CSS</a></li>
```
### css 삽입 방식
    1. <style> 태그를 쓴다.
    2. style 속성을 쓴다.

## 정확한 명칭
```
a {
    color:red;
}
```
| a | Selecter(선택자) |
| color:red; | Declaration(선언) |
| color | Property(속성) |
| red | Value(값) |

## class - 하나로 묶는다
```
<ol>
    <li class="saw">HTML</li>
    <li class="saw">CSS</li>
    <li>JS</li>
</ol>

<style>
    .saw {
        color:gray;
    }
</style>
```
style에서 class 앞에 꼭 .을 찍어줘야 함

```
<ol>
    <li class="saw">HTML</li>
    <li class="saw active">CSS</li>
    <li>JS</li>
</ol>

<style>
    .saw {
        color:gray;
    }
    .active {
        color:red;
    }
</style>
```
css코드 안에서 active가 saw보다 나중에 등장했기 때문에 active의 영향을 받아 폰트는 빨간색이 됨
즉, 가장 최근의 명령이 더 큰 영향력을 가진다

## ID 선택자
```
<ol>
    <li class="saw">HTML</li>
    <li class="saw" id="active">CSS</li>
    <li>JS</li>
</ol>

<style>
    #active {
        color:red;
    }
    .saw {
        color:gray;
    }
</style>
```
.saw가 #active보다 늦게 등장했음에도 폰트는 빨간색이 됨
ID 선택자 > class 선택자 > 태그 선택자
id 선택자는 한 번만 등장해야 함

## 박스 모델
- inline element: 자기 자신의 콘텐츠 크기만큼을 쓰는 태그 `display: inline`
- block element: 화면 전체를 쓰는 태그 `display: block`

### 중복 없애기
```
<style>
    h1 {
        border-width: 5px;
        border-color: red;
        border-style: solid;
    }
    a {
        border-width: 5px;
        border-color: red;
        border-style: solid;
    }
</style>
```
=>
```
<style>
    h1, a {
        border-width: 5px;
        border-color: red;
        border-style: solid;
    }
</style>
```
=>
```
<style>
    h1, a {
        border: 5px solid red;
    }
</style>
```

Margin - 선 바깥쪽
Border - 박스의 선
Padding - 선 안쪽
Content - 내용(문구)
순서로 박스 구성
 - 조정이 필요하다면 개발자모드(F12)를 어떤 태그의 margin, padding 등의 값인지를 시각적으로 확인하면서 수정하면 된다.

## Grid
```
<div id="gird">
    <div>Naviagtion</div>
    <div>Article</div>
</div>
```
```
#grid {
    display: grid;
    grid-template-colums: 150px 1fr;
}
```
div는 block element기 때문에 위아래로 Navigation과 Article이 구성됨
css에 저 두 요소를 적으면 두 단어가 나란히 배치됨 - 다만 나란히 놓기 위해서는 부모태그가 필요하기 때문데 <div>로 감싸줌
첫번째 요소는 150px만큼을 차지하고 두번째 요소가 나머지를 다 차지함
```
#grid {
    display: grid;
    grid-template-colums: 2fr 1fr;
}
```
화면 전체를 3fr이라고했을때 2:1로 나누어짐

### 추가 팁
```
#grid ol {
  color: red;
}
```
이렇게 id선택자 뒤에 태그가 오면 id="grid"를 가진 요소 안에 있는 ol에만 스타일을 적용한드는 의미

## 미디어 쿼리 - 반응형 웹

-현재 화면의 크기를 아는 법: 개발자 도구 연 다음 화면 크기 변경해보면 오른쪽 위에서 화면 크기 확인 가능

화면의 너비가 800px보다 크다면 <div> 태그에 display:none을 설정하는 코드
```
<style>
@media(min-width: 800px) {
    div {
        display: none;
    }
}
</style>
```
## <link>태그

```
<head>
    <title>WEB</title>
    <link rel="stylesheet" href="style.css" />
    <style>
     ...생략...
    </style>
</head>
```
link태그를 통해 html과 css 파일을 연결
rel="stylesheet": 연결하는 파일이 CSS스타일 파일이라는 뜻
